🔢 1. Address Formulas ★ EVERY YEAR▼
Key: w = word size · L = lower bound · N = number of columns · M = number of rows
1D (0-based)
Base + i×w
1D (lower bound L)
Base + (i−L)×w
2D Row Major — C, C++, Java (size M×N, 1-indexed)
Base + [(i−Lr)×N + (j−Lc)] × w
2D Column Major — Fortran, MATLAB (size M×N, 1-indexed)
Base + [(j−Lc)×M + (i−Lr)] × w
3D Row Major (size L×M×N, 0-based)
Base + (i×M×N + j×N + k) × w
Row Major: fix row, sweep cols (j free) · Col Major: fix col, sweep rows (i free)
A[1..5][1..4], base=1000, w=4, Row Major → A[3][4]?
1000 + [(3−1)×4 + (4−1)]×4 = 1000+44 = 1044
Same array, Column Major → A[3][4]?
1000 + [(4−1)×5 + (3−1)]×4 = 1000+68 = 1068
Trap: Negative lower bounds: offset = i−(−5) = i+5. Always subtract Lb from index first.

🔎 5. Searching Algorithms▼
AlgorithmPrerequisiteBestAverageWorst
LinearNoneO(1)O(n)O(n)

Binary Search — safe mid formula
mid = low + (high − low) / 2 ← NOT (low+high)/2 → overflow! Comparisons worst = ⌊log₂n⌋ + 1
Interpolation Search — probe position
pos = low + [(target − A[low]) / (A[high] − A[low])] × (high − low)
Traps: Interpolation worst = O(n) for exponential data · Ternary needs 2 comparisons per step vs 1 for binary → binary preferred for arrays · Ternary used for unimodal function min/max
🛠 6. Key Techniques — When to Use What▼
Two Pointer

When: Sorted array + pair/triplet

How: l=0, r=n−1; squeeze inward

Cost: O(n) time O(1) space

Sliding Window

When: Contiguous subarray of size k

How: Add new − remove old each step

Cost: O(n) vs O(nk) brute force

Prefix Sum

When: Multiple range sum queries

How: P[i]=P[i−1]+A[i]; sum(l,r)=P[r]−P[l−1]

Cost: O(n) build, O(1) query

Pattern → Technique:
Contiguous subarray max/min/sum → Sliding Window or Kadane's
Sorted + pair that hits target → Two Pointer
Many range queries → Prefix Sum
Missing / repeated element → Sum formula n(n+1)/2 or XOR
🔍 7. Key Algorithms — Formula Cards▼

Kadane's — Max Subarray Sum

maxHere = maxSoFar = A[0] for i=1..n-1: maxHere = max(A[i], maxHere+A[i]) maxSoFar = max(maxSoFar, maxHere)
Extend or restart — whichever bigger

Boyer-Moore — Majority (>n/2)

cand=A[0], cnt=1 for i=1..n-1: if A[i]==cand: cnt++ else: cnt-- if cnt==0: cand=A[i],cnt=1 verify cand
Cancel different, keep same

Dutch National Flag — Sort 0,1,2

lo=0, mid=0, hi=n-1 while mid≤hi: 0→swap(lo,mid);lo++;mid++ 1→mid++ 2→swap(mid,hi);hi--
3-pointer single pass O(n) O(1)

Rotation — Reversal Method

Left rotate by k: 1. reverse A[0..k-1] 2. reverse A[k..n-1] 3. reverse A[0..n-1]
3 reversal = 1 rotation O(n) O(1)

Missing Number:

Missing = n(n+1)/2 − actual sum OR XOR 1..n with all elements

Trapping Rain Water:

water[i] = min(maxLeft[i], maxRight[i]) − height[i] Two-pointer O(1) space: if leftMax<rightMax → water+=leftMax−h[left]; left++ else → water+=rightMax−h[right]; right--
🔄 8. Circular Arrays & Queue▼
Core index tricks
Next: (i+1) % n Previous: (i−1+n) % n ← +n avoids negative k forward: (i+k) % n Distance i→j: (j−i+n) % n
Circular Queue (array size n, capacity n−1)
FULL: (REAR+1) % n == FRONT EMPTY: REAR == FRONT Size: (REAR − FRONT + n) % n + 1
Capacity = n−1 (1 slot wasted to tell full from empty)
Classic Trap: Full = (REAR+1)%n==FRONT · Empty = REAR==FRONT — DON'T swap these!
Circular queue size=6, front=2, rear=5 → elements present?
(5−2+6)%6 + 1 = 3+1 = 4
📈 9. Amortized Analysis — Dynamic Arrays▼
Key idea: Occasional expensive op is rare enough that average (amortized) cost = O(1).
Doubling strategy: resize cost = 1+2+4+...+n = 2n Total cost for n inserts = n + 2n = 3n = O(n) Amortized per insert = O(1)
Fixed-k growth → O(n) amortized · Doubling → O(1) amortized
Trap: Amortized O(1) ≠ actual O(1). Individual resize is O(n). Total for n ops = O(n).
⚠️ 10. Complete Trap Master List▼
Array & Operations:
• Delete middle even if index known = O(n) (shifting) — NOT O(1)
• Arrays cache-friendly; Linked Lists are NOT
• mid = low+(high−low)/2 — never (low+high)/2 — overflow!

Sorting:
• Quick Sort worst = O(n²) on sorted input (always)
• Merge Sort NOT in-place → O(n) space
• Heap Sort in-place but NOT stable
• Selection Sort NOT stable · best for minimising swaps
• Lower bound for ANY comparison sort = Ω(n log n)
• Counting/Radix bypass Ω(n log n) — they're non-comparison based

Searching:
• Interpolation worst = O(n) on non-uniform data
• Ternary = more comparisons per step than Binary

Circular Queue:
• Full: (REAR+1)%n==FRONT · Empty: REAR==FRONT (don't swap!)
• Capacity = n−1, NOT n
• (i−1+n)%n not (i−1)%n — prevents negative index

Matrices:
• Lower triangular non-zeros = n(n+1)/2 (includes diagonal)
• WITHOUT diagonal = n(n−1)/2
• Tri-diagonal = 3n−2

Amortized:
• Amortized O(1) ≠ actual O(1) per operation
📝 11. 20 PYQ MCQs — Cover & Solve First! ★★★▼
Q1. Which sort has O(n log n) in ALL three cases?
(A) Quick (B) Merge (C) Heap (D) Both B & C
D — Merge & Heap always O(n log n). Quick degrades to O(n²).
Q2. Best sort to minimise number of swaps?
(A) Bubble (B) Insertion (C) Selection (D) Merge
C — Selection Sort: at most O(n) swaps.
Q3. Interpolation Search average case on uniform data?
(A) O(log n) (B) O(log log n) (C) O(n) (D) O(√n)
B — O(log log n) avg; O(n) worst on non-uniform.
Q4. Circular queue size=7, front=3, rear=6 → elements?
(A) 3 (B) 4 (C) 5 (D) 6
B — (6−3+7)%7+1 = 10%7+1 = 4.
Q5. Which sort is NOT stable?
(A) Bubble (B) Merge (C) Heap (D) Insertion
C — Heap Sort unstable. BIM-CR = stable.
Q6. Recurrence T(n)=2T(n/2)+n solves to?
(A) O(n) (B) O(n log n) (C) O(n²) (D) O(log n)
B — Master Theorem: a=b=2, log₂2=1 → O(n log n).
Q7. Amortized cost of n insertions in a doubling dynamic array?
(A) O(n²) (B) O(n log n) (C) O(n) (D) O(1)
C — Total = O(n). Per insert amortized = O(1), but total = O(n).
Q8. Worst comparisons for Binary Search on 64 elements?
(A) 5 (B) 6 (C) 7 (D) 8
C — ⌊log₂64⌋+1 = 6+1 = 7.
Q9. Find majority element in O(n) O(1) — which algorithm?
(A) Kadane's (B) Boyer-Moore (C) DNF (D) Floyd's
B — Boyer-Moore Voting Algorithm.
Q10. A[i] at address 200+4i. Address of A[5]?
(A) 215 (B) 220 (C) 225 (D) 224
B — 200+4×5 = 220.
Q11. Best sort for a linked list?
(A) Quick (B) Heap (C) Merge (D) Insertion
C — Merge Sort needs no random access. Quick Sort is O(n²) on linked lists.
Q12. Bubble Sort best case O(n) occurs when?
(A) Reverse order (B) Already sorted (C) All equal (D) Both B & C
D — Optimised bubble (swap flag) exits in O(n) for sorted or equal arrays.
Q13. B[1..8][1..8], w=4, col major, base=1000 → B[3][5]?
(A) 1104 (B) 1112 (C) 1108 (D) 1136
D — 1000+[(5−1)×8+(3−1)]×4 = 1000+[32+2]×4 = 1136.
Q14. Quick Sort uses which paradigm?
(A) Greedy (B) DP (C) Divide & Conquer (D) Backtracking
C — Quick, Merge, Binary Search all use Divide & Conquer.
Q15. Minimum comparisons to find maximum in unsorted array of n?
(A) n (B) n−1 (C) n/2 (D) log n
B — Exactly n−1 comparisons needed.
Q16. Which is NOT in-place?
(A) Heap (B) Quick (C) Merge (D) Bubble
C — Merge Sort needs O(n) auxiliary space.
Q17. Row major A[1..n][1..n] — which access is cache-efficient?
(A) A[j][i] (B) A[i][j] (C) Same (D) Depends
B — Row-wise exploits cache lines. Column-wise causes cache misses.
Q18. Lower bound for comparison-based sorting?
(A) O(n) (B) O(n log n) (C) O(n²) (D) O(log n)
B — Decision tree proof: Ω(n log n) in worst case.
Q19. How do Counting/Radix beat O(n log n)?
(A) Clever comparisons (B) Non-comparison based (C) Hashing (D) D&C
B — They use digit/count info directly — bypass the Ω(n log n) lower bound.
Q20. Circular queue capacity n−1, array size n, REAR=FRONT=0. Full condition?
(A) (REAR+1)%n==FRONT (B) REAR==FRONT (C) REAR==(FRONT+1)%n (D) (FRONT+1)%n==REAR
A — REAR chases FRONT: (REAR+1)%n==FRONT means full. REAR==FRONT means empty.
