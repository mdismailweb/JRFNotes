📌 1. Definition & Memory Model ▼

An array is a linear, homogeneous data structure that stores elements at contiguous memory locations. Each element is accessed via an index in O(1) time.

Key Property: Base Address + Offset → Direct access to any element. This is why random access is O(1).
Advantages
  • Random access — O(1)
  • Cache-friendly (spatial locality)
  • Simple & fast for indexed access
  • Used to build Stack, Queue, Heap
Disadvantages
  • Fixed size (static arrays)
  • Insert/Delete in middle — O(n)
  • Wastage if pre-allocated too large
  • Searching unsorted array — O(n)
Memory Address Formula (1D)
Address of A[i] = Base + i × size_of_element
Memorizer: "BASE + INDEX × ELEMENT SIZE" — the golden triangle of array addressing.

🔢 3. Address Calculation Formulas ★ High Yield ▼
UGC NET ALERT: Address formula questions appear almost every year. Master all cases below.

1D Array — 0-based indexing:

Addr(A[i]) = Base + i × w

1D Array — 1-based indexing (Lower Bound = L):

Addr(A[i]) = Base + (i − L) × w

2D Array — Row Major Order (C, C++):

0-indexed, size M×N
Addr(A[i][j]) = Base + (i × N + j) × w
1-indexed (LB row=Lr, LB col=Lc)
Addr(A[i][j]) = Base + [(i−Lr) × N + (j−Lc)] × w

2D Array — Column Major Order (Fortran, MATLAB):

0-indexed, size M×N
Addr(A[i][j]) = Base + (j × M + i) × w
1-indexed
Addr(A[i][j]) = Base + [(j−Lc) × M + (i−Lr)] × w
Memorizer: Row Major = "Row first, fix row then sweep columns (j free)"; Column Major = "Column first, fix col then sweep rows (i free)"

3D Array — Row Major (size L×M×N):

Addr(A[i][j][k]) = Base + (i×M×N + j×N + k) × w
Q: An array A[1..5][1..4] with base 1000 and word size 4 bytes stored in row major. Find address of A[3][4].
N = 4 columns. Addr = 1000 + [(3−1)×4 + (4−1)] × 4 = 1000 + [8+3]×4 = 1000 + 44 = 1044
Q: Same array in column major. Find address of A[3][4].
M = 5 rows. Addr = 1000 + [(4−1)×5 + (3−1)] × 4 = 1000 + [15+2]×4 = 1000 + 68 = 1068

🛠 5. Key Techniques — Two Pointer, Sliding Window, Prefix Sum ▼

A. Two Pointer Technique

Use two indices (left, right) moving toward each other or in the same direction. Reduces O(n²) brute force to O(n).

Pattern: Sorted array + "pair with sum = target" → left=0, right=n-1.
If A[l]+A[r] < target: l++  |  If > target: r--  |  If = target: found!
Memorizer: "Squeeze from both ends on sorted arrays"
Q: Find if a sorted array has a pair with sum = 17. Array: [1, 3, 6, 8, 9, 11]
l=0(1), r=5(11)→12<17, l++; l=1(3),r=5(11)→14<17, l++; l=2(6),r=5(11)→17=17 → Yes, (6,11)

B. Sliding Window Technique

Maintain a "window" of fixed or variable size sliding over the array. Avoids recomputing the entire window from scratch.

Fixed Window of size k — Maximum Subarray Sum
1. Compute sum of first k elements → window_sum 2. For i = k to n-1: window_sum += A[i] − A[i−k] max_sum = max(max_sum, window_sum)
Complexity: O(n) instead of O(n×k) brute force.
Memorizer: "Add new, remove old — the window slides in O(1) per step"

C. Prefix Sum Technique

Precompute cumulative sums so that any subarray sum can be found in O(1).

Build prefix array
P[0] = A[0] P[i] = P[i−1] + A[i]   Sum(A[l..r]) = P[r] − P[l−1] (where P[−1] = 0)
Use case: Multiple range-sum queries → precompute once O(n), answer each query in O(1).
Q: Array = [2, 4, 6, 8, 10]. Find sum of A[1..3] using prefix sum (0-indexed).
P = [2, 6, 12, 20, 30]. Sum(1..3) = P[3] − P[0] = 20 − 2 = 18
🔍 6. Important Algorithms on Arrays ▼

Kadane's Algorithm — Maximum Subarray Sum

max_ending_here = max_so_far = A[0] For i=1 to n-1: max_ending_here = max(A[i], max_ending_here + A[i]) max_so_far = max(max_so_far, max_ending_here) Return max_so_far
Complexity: O(n) time, O(1) space. Handles all-negative arrays if initialized to A[0].
Memorizer: "Either extend the previous subarray or start fresh — whichever is bigger"

Dutch National Flag (DNF) — Sort 0s, 1s, 2s

Pointers: low=0, mid=0, high=n-1 While mid ≤ high: A[mid]=0 → swap(A[low],A[mid]); low++; mid++ A[mid]=1 → mid++ A[mid]=2 → swap(A[mid],A[high]); high--
Complexity: O(n) time, O(1) space. Single pass.

Boyer-Moore Voting Algorithm — Majority Element (>n/2 occurrences)

candidate = A[0], count = 1 For i=1 to n-1: if A[i] == candidate: count++ else: count--; if count=0: candidate=A[i], count=1 Verify candidate by counting → if count > n/2: return candidate
Memorizer: "Cancel different, keep same — survivor is candidate majority"

Array Rotation — Reversal Algorithm

Left rotate by k positions: 1. Reverse A[0..k-1] 2. Reverse A[k..n-1] 3. Reverse entire array A[0..n-1]
Complexity: O(n) time, O(1) space.

Binary Search (on sorted array)

low=0, high=n-1 While low ≤ high: mid = low + (high−low)/2 ← avoids overflow vs (low+high)/2 if A[mid]=target: return mid if A[mid]<target: low=mid+1 else: high=mid-1 return -1
Trap: Use mid = low + (high−low)/2 NOT (low+high)/2 to prevent integer overflow!

Finding Missing Number in [1..n]

Expected sum = n×(n+1)/2 Missing = Expected − Actual Sum of array   XOR Method (handles overflow): XOR all indices 1..n with all array elements

🔷 8. Sparse & Special Matrices ★ UGC NET Favourite ▼
UGC NET ALERT: Sparse matrix storage and diagonal matrix formulas are frequently tested.

Sparse Matrix: A matrix where most elements are zero. Storing all n² elements wastes space. Represented as triplet <row, col, value>.

Non-zero elements stored: each as (row, col, value) triplet Space: O(k) where k = number of non-zero elements (vs O(n²) for dense)

Lower Triangular Matrix (n×n): Elements stored only at positions where row ≥ col.

Non-zero elements = n(n+1)/2 Row-major storage position of A[i][j] (i≥j, 1-indexed): Index = i(i−1)/2 + j − 1 (0-based array position)

Upper Triangular Matrix:

Index of A[i][j] (i≤j, 1-indexed): = (i−1)(2n−i)/2 + j − i (row major)

Diagonal Matrix (only diagonal elements non-zero):

Non-zero = n elements only. A[i][i] stored at index i. A[i][j] = 0 if i ≠ j

Tri-diagonal Matrix (band matrix, bandwidth=3):

Non-zero elements on: main diagonal + one above + one below Total non-zero = 3n − 2 Index of A[i][j] = 2(i−1) + (j−i) + 1 (1-indexed, row major)
Q: A lower triangular matrix A[1..4][1..4] is stored row-wise starting at address 100, word size = 2. Find address of A[3][2].
Index (0-based) = 3(3−1)/2 + 2 − 1 = 3 + 1 = 4. Address = 100 + 4×2 = 108
📝 9. UGC NET PYQ-Style Questions ▼
Q1: An array A[−5..5][−3..3] uses word size 4 bytes, row major, base address 100. Find address of A[2][2].
Rows: −5 to 5 → 11 rows. Cols: −3 to 3 → 7 cols (N=7).
Row offset = 2−(−5) = 7. Col offset = 2−(−3) = 5.
Addr = 100 + (7×7 + 5) × 4 = 100 + 54×4 = 100 + 216 = 316
Q2: The time complexity of inserting an element at a specific position in an array of size n is:
O(n) — because up to n elements may need to be shifted.
Q3: Which algorithm finds the maximum subarray sum in O(n) time and O(1) space?
Kadane's Algorithm
Q4: A 2D array A[1..m][1..n] stored in column major. What is the address of A[i][j]? (Base = B, size = w)
Addr = B + [(j−1)×m + (i−1)] × w = B + ((j−1)m + i−1) × w
Q5: Number of elements in lower triangular matrix of size n×n including diagonal?
n(n+1)/2
Q6: An array has elements [3, 1, 4, 1, 5, 9, 2, 6]. What does Kadane's algorithm return?
Running max: 3→4→8→9→14→23→25→31. 31 (entire array is the max subarray here since all positive)
Q7: A circular buffer of size n is implemented using an array. How do you efficiently compute (front+1) % n?
Using modulo arithmetic: (front + 1) % n — wraps index back to 0 after reaching n−1.
Q8: Which traversal order is better for 2D array cache performance — row-wise or column-wise?
Row-wise — because arrays in C/C++ are stored in row-major order, so row-wise access exploits spatial locality and cache lines.

Key Algorithm Walkthroughs:

Insertion Sort — How it works:

For i = 1 to n-1: key = A[i] j = i - 1 While j >= 0 AND A[j] > key: A[j+1] = A[j] j-- A[j+1] = key → Builds sorted portion from left, inserting each new element in correct position

Quick Sort — Partition Logic (Lomuto scheme):

pivot = A[high] i = low - 1 For j = low to high-1: if A[j] <= pivot: i++, swap(A[i], A[j]) swap(A[i+1], A[high]) return i+1 ← pivot's final position

Merge Sort — Divide & Conquer:

MergeSort(A, l, r): if l < r: mid = (l+r)/2 MergeSort(A, l, mid) MergeSort(A, mid+1, r) Merge(A, l, mid, r) ← O(n) merge step Recurrence: T(n) = 2T(n/2) + O(n) → O(n log n) by Master Theorem
Master Theorem Quick Apply: T(n) = aT(n/b) + f(n)
If f(n) = O(n^log_b(a)): T(n) = O(n^log_b(a) × log n)
Merge Sort: a=2, b=2, f(n)=O(n) → log_2(2)=1 → O(n log n) ✓

🔎 12. Searching Algorithms — Binary, Interpolation, Ternary ▼

Binary Search (recap + recurrence):

Recurrence: T(n) = T(n/2) + O(1) → O(log n) Iterations needed: ⌊log₂n⌋ + 1 Works only on SORTED arrays.

Interpolation Search:

Like binary search but uses the value to estimate where the target might be — better for uniformly distributed sorted data.

Probe Position Formula
pos = low + [(target - A[low]) / (A[high] - A[low])] × (high - low)   If A[pos] == target → found If A[pos] < target → low = pos + 1 If A[pos] > target → high = pos - 1
BestAverageWorst
Interpolation SearchO(1)O(log log n)O(n)
Key condition: Works well ONLY when data is uniformly distributed. Worst case O(n) if data is exponentially distributed (e.g., 1,2,4,8,16...).
Memorizer: "Interpolation = Smart Binary — jumps to estimated position instead of always mid"

Ternary Search:

Divides array into 3 parts instead of 2. Used to find maximum/minimum of a unimodal function.

mid1 = low + (high - low) / 3 mid2 = high - (high - low) / 3   If A[mid1] == target → return mid1 If A[mid2] == target → return mid2 If target < A[mid1] → search [low, mid1-1] If target > A[mid2] → search [mid2+1, high] Else → search [mid1+1, mid2-1]   Time Complexity: O(log₃ n) = O(log n / log 3)
Important: Ternary Search makes MORE comparisons than Binary Search per iteration (2 vs 1), so Binary Search is preferred for array search. Ternary Search shines for unimodal functions.

🔄 13. Circular Arrays — Modular Indexing Tricks ★ ▼
UGC NET Context: Circular arrays are used to implement Circular Queue. Modular arithmetic questions are common in queue-based MCQs.

A circular array treats the array as if the end wraps around to the beginning. Index arithmetic uses modulo operator.

Core Modular Tricks
Next index: (i + 1) % n Previous index: (i - 1 + n) % n ← +n prevents negative modulo Move k steps forward: (i + k) % n Move k steps backward: (i - k + n) % n   Distance from i to j (forward): (j - i + n) % n
Trap: (i - 1) % n in some languages gives negative result for i=0. Always write (i - 1 + n) % n to be safe.

Circular Queue using Array (size n):

front = rear = -1 (empty state)   Enqueue: if (rear+1)%n == front → FULL (overflow) else: rear = (rear+1)%n; Q[rear] = value   Dequeue: if front == -1 → EMPTY (underflow) else: value = Q[front]; front = (front+1)%n   Size of queue = (rear - front + n) % n + 1 Max elements storable = n - 1 (one slot kept empty to distinguish full vs empty)
Memorizer: "Circular Queue wastes 1 slot — capacity = n−1 elements in array of size n"
Q: A circular queue of size 6 has front=2, rear=5. How many elements are in it?
Size = (rear - front + n) % n + 1 = (5 - 2 + 6) % 6 + 1 = 9%6 + 1 = 3 + 1 = 4 elements
Q: In a circular array of size 8, current index = 7. What is the next index?
(7 + 1) % 8 = 8 % 8 = 0 (wraps to beginning)

Maximum Circular Subarray Sum:

Answer = max of: Case 1: Normal max subarray sum (Kadane's) — subarray doesn't wrap Case 2: Total sum − min subarray sum — subarray wraps around   If all elements negative → answer = max element (Case 2 doesn't apply)
📈 14. Amortized Analysis — Dynamic Arrays Deep Dive ▼

What is Amortized Analysis? It measures the average cost per operation over a sequence of operations, even if individual operations are occasionally expensive.

Key Insight: A single operation may be O(n) but if it happens rarely enough, the average (amortized) cost across all operations is still O(1).

Dynamic Array Doubling — The Classic Example:

Start: array of size 1 When full: allocate 2× size, copy all elements, insert new element   Insert sequence cost: Insert 1: cost 1 Insert 2: resize (cost 1 copy) + insert → cost 2 Insert 3: resize (cost 2 copies) + insert → cost 3 Insert 4: just insert → cost 1 Insert 5: resize (cost 4 copies) + insert → cost 5 ...   Total cost for n inserts = n + (1 + 2 + 4 + 8 + ... up to n) = n + 2n = 3n = O(n) Amortized cost per insert = O(n)/n = O(1)
Memorizer: "Doubling makes resize rare — total resize cost = 2n, so amortized O(1) per insert"
Growth StrategyAmortized InsertSpace Waste
Double size (×2)O(1)Up to 50%
Add fixed k slotsO(n)Minimal
Triple size (×3)O(1)Up to 67%
Three methods of Amortized Analysis:
1. Aggregate method — compute total cost / n operations
2. Accounting method — assign "credits" to cheap ops, spend on expensive ones
3. Potential method — define a potential function Φ; amortized cost = actual + ΔΦ
Q: A dynamic array starts with size 1 and doubles when full. What is the total number of element copies after inserting 16 elements?
Copies happen at sizes: 1,2,4,8 → copies = 1+2+4+8 = 15. Plus 16 inserts = 31 total ops. Amortized per insert ≈ 31/16 ≈ ~2 = O(1)
🧩 15. Advanced Problems — Trapping Rain Water & Stock Buy & Sell ▼

Trapping Rain Water:

Given an array of bar heights, find total water trapped between bars after rain.

For each index i:
water[i] = min(maxLeft[i], maxRight[i]) − height[i] Total = sum of water[i] for all i   Step 1: Build maxLeft[i] = max(height[0..i]) → left to right pass Step 2: Build maxRight[i] = max(height[i..n-1]) → right to left pass Step 3: water = Σ max(0, min(maxL[i], maxR[i]) − h[i])   Time: O(n) Space: O(n)   Two-Pointer Optimization → O(n) time, O(1) space: Use left, right pointers; track leftMax, rightMax If leftMax < rightMax: water += leftMax − h[left]; left++ Else: water += rightMax − h[right]; right--
Memorizer: "Water at any bar = min of tallest walls on both sides minus bar height"

Stock Buy & Sell — Variants:

Single Transaction (buy once, sell once)

Track minPrice seen so far. At each day: profit = price − minPrice. Answer = max profit seen.

Time: O(n), Space: O(1)
Multiple Transactions (unlimited)

Add all positive differences: if A[i] > A[i-1], add A[i]−A[i-1] to profit.

Time: O(n), Space: O(1)
At most 2 Transactions

4-variable DP: track buy1, sell1, buy2, sell2 states. Single pass O(n).

Time: O(n), Space: O(1)
At most K Transactions

DP table dp[k][n]. dp[i][j] = max profit using i transactions up to day j.

Time: O(kn), Space: O(kn)
📝 16. 20 PYQ-Style MCQs with Full Solutions ★★★ ▼
Exam Format: Each question has 4 options. The correct answer and explanation follow. Cover the answer and solve first!
Q1: Which sorting algorithm has O(n log n) in all cases (best, average, worst)?
(A) Quick Sort   (B) Merge Sort   (C) Heap Sort   (D) Both B and C
Answer: (D) — Merge Sort and Heap Sort both guarantee O(n log n) in all cases. Quick Sort degrades to O(n²) in worst case.
Q2: Which sorting algorithm is best suited when the number of swaps must be minimized?
(A) Bubble Sort   (B) Insertion Sort   (C) Selection Sort   (D) Merge Sort
Answer: (C) — Selection Sort makes at most O(n) swaps, the minimum among all comparison-based sorts.
Q3: What is the time complexity of Interpolation Search on a sorted array with uniformly distributed data?
(A) O(log n)   (B) O(log log n)   (C) O(n)   (D) O(√n)
Answer: (B) — Average case is O(log log n) for uniformly distributed sorted arrays. Worst case is O(n).
Q4: A circular queue is implemented using an array of size 7. Currently front=3, rear=6. How many elements are present?
(A) 3   (B) 4   (C) 5   (D) 6
Answer: (B) — Size = (rear − front + n) % n + 1 = (6−3+7)%7 + 1 = 10%7 + 1 = 3+1 = 4
Q5: Which of the following sorting algorithms is NOT stable?
(A) Bubble Sort   (B) Merge Sort   (C) Heap Sort   (D) Insertion Sort
Answer: (C) — Heap Sort is not stable. Bubble, Merge, and Insertion sorts are stable. Remember: BIM-CR is stable.
Q6: The recurrence relation for Merge Sort is T(n) = 2T(n/2) + n. What is its solution?
(A) O(n)   (B) O(n log n)   (C) O(n²)   (D) O(log n)
Answer: (B) — By Master Theorem: a=2, b=2, f(n)=n=O(n^log₂2)=O(n¹) → Case 2 → O(n log n)
Q7: In a dynamic array that doubles its size when full, what is the amortized cost of n insertions?
(A) O(n²)   (B) O(n log n)   (C) O(n)   (D) O(1)
Answer: (C) — Total cost = O(n). Amortized per insertion = O(n)/n = O(1), but total for n ops is O(n).
Q8: What is the worst-case number of comparisons in Binary Search on an array of 64 elements?
(A) 5   (B) 6   (C) 7   (D) 8
Answer: (C) — ⌊log₂(64)⌋ + 1 = 6 + 1 = 7 comparisons.
Q9: Which algorithm is used to find the majority element (appears more than n/2 times) in O(n) time and O(1) space?
(A) Kadane's Algorithm   (B) Boyer-Moore Voting   (C) Dutch National Flag   (D) Floyd's Algorithm
Answer: (B) — Boyer-Moore Voting Algorithm finds the majority element in a single pass with O(1) extra space.
Q10: An array A[0..n-1] is stored in memory. Element A[i] is at address 200+4i. Element A[5] is at:
(A) 215   (B) 220   (C) 225   (D) 224
Answer: (B) — 200 + 4×5 = 200 + 20 = 220
Q11: Which sort is most efficient for sorting a linked list?
(A) Quick Sort   (B) Heap Sort   (C) Merge Sort   (D) Insertion Sort
Answer: (C) — Merge Sort works naturally on linked lists (no random access needed). Quick Sort requires random access, making it O(n²) on linked lists.
Q12: The best case time complexity of Bubble Sort is O(n). Under what condition does this occur?
(A) Array is in reverse order   (B) Array is already sorted   (C) Array has all equal elements   (D) Both B and C
Answer: (D) — With the optimized version (flag to detect no swaps), Bubble Sort runs in O(n) for already sorted or all-equal arrays.
Q13: A 2D array B[1..8][1..8] stores integers (4 bytes). It is stored in column major. Base address = 1000. Address of B[3][5]?
(A) 1104   (B) 1112   (C) 1108   (D) 1136
Answer: (D) — Col major: Addr = 1000 + [(5-1)×8 + (3-1)] × 4 = 1000 + [32+2]×4 = 1000+136 = 1136
Q14: Quick Sort uses which design paradigm?
(A) Greedy   (B) Dynamic Programming   (C) Divide and Conquer   (D) Backtracking
Answer: (C) — Quick Sort, Merge Sort, and Binary Search all use Divide and Conquer.
Q15: What is the minimum number of comparisons required to find the maximum element in an unsorted array of n elements?
(A) n   (B) n-1   (C) n/2   (D) log n
Answer: (B) — You need exactly n−1 comparisons: compare each element to the current maximum.
Q16: Which of the following is NOT an in-place sorting algorithm?
(A) Heap Sort   (B) Quick Sort   (C) Merge Sort   (D) Bubble Sort
Answer: (C) — Merge Sort requires O(n) auxiliary space for the temporary array during merging.
Q17: For an array A[1..n][1..n] stored in row major, which access pattern is more cache-efficient?
(A) for i: for j: access A[j][i]   (B) for i: for j: access A[i][j]   (C) Both equal   (D) Depends on n
Answer: (B) — Row-major storage means A[i][j] and A[i][j+1] are adjacent in memory. Row-wise access (B) exploits cache lines; column-wise (A) causes cache misses.
Q18: The lower bound on comparison-based sorting algorithms is:
(A) O(n)   (B) O(n log n)   (C) O(n²)   (D) O(log n)
Answer: (B) — Proven via decision tree argument: any comparison-based sort needs at least Ω(n log n) comparisons in worst case. Merge Sort and Heap Sort achieve this lower bound.
Q19: Which technique is used by Counting Sort and Radix Sort to achieve better than O(n log n)?
(A) They use comparisons cleverly   (B) They do not use comparisons — they use counting/digit info   (C) They use hashing   (D) They use divide and conquer
Answer: (B) — Counting/Radix sorts are non-comparison based. They bypass the O(n log n) lower bound by using element values/digits directly, not comparisons.
Q20: A sorted array of 1000 elements. Interpolation Search vs Binary Search — which makes fewer probes on average if data is uniform?
(A) Binary Search   (B) Interpolation Search   (C) Both same   (D) Depends on target value
Answer: (B) — Interpolation Search: O(log log 1000) ≈ O(log 10) ≈ 3–4 probes. Binary Search: O(log 1000) ≈ 10 probes. Interpolation wins for uniform data.

Key Exam Traps to Avoid:
• Quick Sort worst case = O(n²) on sorted arrays

• Circular Queue capacity = n−1 (not n)
• Binary Search: mid = low+(high−low)/2 to avoid overflow

• Amortized O(1) insert ≠ actual O(1) (individual resize is O(n))
